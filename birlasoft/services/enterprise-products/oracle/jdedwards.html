Tools/Accelerators
ProVue - An automated comprehensive JD Edwards monitoring tool
Overview
Specialized E1 application monitoring solution. Built for E1 with scope for customization and flexibility to suit your environment.
Challenges:
  • Popular monitoring tools available in the market does server monitoring only.
  • Currently there are no out of box tools which provides application monitoring and alerts
Features:
  • Built on industry leading machine data platform
  • Customizable dashboards, alerts, and metrics
  • Designed from the ground up for E1
  • Real-time and predictive monitoring with trend analysis
  • Real end-user behavior and performance monitoring
  • Monitor infrastructure KPIs, JDE KPIs, UBEs, logs, with metric history
  • 24x7 automated monitoring and portal access
  • Hosted and managed by us on Oracle Cloud Infrastructure
Benefits:
  • Reduced incident resolution time
    With all essential and related information provided on a single pane of glass and the ability to traverse multiple servers and hundreds of log files almost instantaneously, a CNC administrator can quickly solve the problem at hand.
  • Proactive monitoring
    Reduce unexpected downtime by identifying and resolving issues before they impact the business with ProVue constantly monitoring the E1 landscape.
  • Flexible
    Built for JDE, but flexible enough to monitor non-JDE systems, third party integrations, or to incorporate your existing monitoring scripts into its data feed.
  • Data driven planning
    Historical and real-time data points are stored for a minimum of 12 months which helps in making an informed decision when planning for JDE server expansion, migrations, or upgrades.
  • Detailed user insights
    Real-time end user behavior analytics to determine usage patterns, peak loads, most used applications, and historical sessions. Go from running surface tests from a single machine to capturing results from your entire JDE user base.

VerGen - For automated version copy and update
Overview
VerGen provides features to generate versions based on an excel input or synchronize processing option (P.O.), data selection (D.S.) or both based from a template version.
Challenges:
  • Rollout/upgrade require versions to be created and updated
  • Manual creation/updating of these versions involve huge manual efforts
  • No provision to backup previous version details (D.S. and P.O.)
Features:
  • Consistent and user-friendly UI
  • Mass copy/update of versions
  • Download version details (D.S. and P.O.) to Excel
  • Upload version details from Excel once the user makes the changes required
  • Integrated with the Object Management Workbench (OMW)
  • Maintaining an audit trail
Benefits:
  • Audit trail of previous version details (DS and PO)
  • Excel download and upload Facility
  • 70% reduction in development time
  • Web based tool and does not need a FAT client

ProDart - For processing options and data selection compare and updates
Overview
ProDart provides functionality to compare and document processing option (P.O.) and data selection (D.S.) values across instances or environments on the same instance.
Challenges
  • P.O. and D.S. are important blocks and need to be verified when upgrading to a newer release or applying an ESU/Project
  • P.O. and D.S. can be modified over time in your instance and may not be consistent across environments
  • Difficult to document your current configuration
  • A manual review of the P.O. and D.S. values across instances, environments is tedious, labor intensive and prone to human errors
Features:
  • ProDart provides P.O. and D.S. values in a convenient and easily readable format
  • ProDart will compare the P.O. and D.S. values across
    • Instances
    • Environments
    • Versions
  • ProDart can document the P.O. and D.S. values in the ProDart environment
Benefits:
  • 80% reduction in time
  • Accurate results
  • Cross-instance comparison
  • 100% elimination of manual errors and rework
  • Automated process

iLink - An integration adapter for JD Edwards with PLM, MDM or Oracle SaaS products
Overview
Pre-built adapter to provide synchronization of master data between JD Edwards E9.2 and PLM, MDM or Oracle SaaS products (PDH, CPQ).
Features:
  • Modular design - Data Enrichment, Data Validation and File Processor blocks
  • Mode of operation - Priority, Batch and Bulk mode of operations
  • Number of threads - Multi-threaded operation which is configurable
  • Error dashboard with the feature to re-submit failed transactions
Benefits:
  • Pre-built adapter with scope to customize and tailor as per your environment
  • Keep master data in sync between JD Edwards and Third-Party products
  • Built using JD Edwards tools

Testing X-Ref - A multi-level object cross reference utility
Overview
Enhanced cross reference utility which provides single level or multi-level lookups which can either be used for top down or bottom up search.
Challenges
  • JD Edwards Cross-reference provides a one is one relationship only
  • Time consuming to find all dependencies of an object
  • Increased complexity in case of Business Functions
Features:
  • Minimum Pre-requisites and built within the JD Edwards toolset
  • Variants of Testing X-Ref
    • Single level builds a complete list of dependent objects and can be deployed for any type of objects
    • Multi-level builds a complete tree listing of dependent objects and can be deployed for business functions
    • DD Items builds a complete list of objects which utilize the DD Item selected
Benefits:
  • Reduce risk associated with unknown factors
  • 80% reduction in time as compared to manual approach
  • Automated process
  • Accelerate impact assessment for upgrades, code current or ESU

  • Code auto-retrofit for CBSFNs and business services
Benefits:
  • 30% reduction in retrofit cycle
  • Comprehensive error reporting
  • Designed for E1
  • Automated process

DRT – Data replication tool
Overview
Data replication tool provides excel templates and conversion programs to upload data from non-JD Edwards instances to JD Edwards E920.
Challenges:
  • Data migration from non-JD Edwards instances or consolidation of multiple JD Edwards instances pose an issue
  • These needs mapping templates and custom upload programs to be developed
  • Z/EDI processor provide limited functionalities and are not available for all the standard E1 tables
Features:
  • Pre-defined Excel based templates with rules defined, export and import functionality

  • Auto generation of success and failure audit reports
  • Upload programs for master data, opening balances and open orders

  • User friendly Excel based templates and dashboard
Benefits:
  • Accelerate the data migration phase
  • 100% elimination of manual errors & rework
  • 80% reduction in development time
  • Automated process
